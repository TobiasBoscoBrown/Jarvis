How It Works

Jarvis runs in the background, always listening for "Hey Jarvis". When it hears the wake word, it records your voice command, transcribes it with OpenAI Whisper, and routes it to Claude Cowork, Claude Code, or any custom action you define.

Setup Guide

No API Keys Needed for Wake Word!

Jarvis uses OpenWakeWord — a completely free, open-source wake word engine. No sign-ups, no keys, no trials. The only key you need is your OpenAI Whisper API key for speech-to-text (already configured in config.json).

Dependencies (Already Installed!)

All Python packages (openwakeword, pyaudio, openai, numpy, onnxruntime) are already installed on your machine. If you ever need to reinstall:

cd Desktop\Jarvis setup.bat

Launch Jarvis

Double-click START_JARVIS.bat on your Desktop, or run from terminal:

cd Desktop\Jarvis python jarvis_core.py

Say "Hey Jarvis"

The console will show a green indicator when the wake word is detected. Speak your command naturally — Jarvis handles the rest.

Zero setup friction: Wake word detection is completely free via OpenWakeWord. On first launch, it auto-downloads a tiny ~5MB model. After that, everything works offline — no accounts, no keys, no limits.

Create Your Own Voice Commands

There are two ways to add custom commands — edit the JSON file directly, or use the interactive command manager.

Method 1: Edit custom_commands.json

Open Desktop\Jarvis\commands\custom_commands.json and add an entry:

{ "commands": [ { "name": "deploy to production", "triggers": [ "deploy to production", "ship it", "deploy prod" ], "action": "prompt_claude_code", "description": "Tells Claude Code to deploy" }, { "name": "open my project", "triggers": [ "open my project", "open viral engine" ], "action": "chain:launch code; wait 2; focus Visual Studio Code", "description": "Opens VS Code" } ] }
Hot reload: After editing the file, just say "Hey Jarvis, reload commands" — no restart needed!

Method 2: Command Manager CLI

Run the interactive manager for a guided experience:

cd Desktop\Jarvis python jarvis_manager.py

This lets you add, edit, delete, and backup commands through a menu interface.

Custom Chain Actions

Use chain:... for advanced multi-step automations. These run cc.py commands in sequence:

chain:launch notepad; wait 1; type Hello

chain:focus Chrome; key ctrl+t; type github.com; key enter

Any cc.py action works inside chains — clicks, types, screenshots, waits, window management.

Pro Tips

Speak naturally

You don't need exact phrases. "Hey Jarvis, can you prompt Cowork to fix the auth bug" works just as well as "prompt cowork fix auth bug". Whisper handles natural speech beautifully.

Voice-to-type anywhere

Click into any text field (email, Slack, docs, code editor), then say "Hey Jarvis, type [your message]". Instant voice dictation with near-zero friction.

Chain commands for workflows

Create custom commands with chain:... actions to automate entire workflows. Open apps, navigate, type, click — all from one voice command.

Fallback behavior

If Jarvis doesn't recognize a specific command pattern, it automatically sends your speech as a prompt to Claude Cowork. So you can just talk to it freely.

Check the logs

Every command is logged to logs/jarvis_YYYYMMDD.log. If something isn't working right, the logs show exactly what was heard, how it was routed, and what happened.

Sensitivity tuning

If Jarvis triggers too easily (or not enough), adjust oww_threshold in config.json. Range is 0.0 to 1.0 — higher = harder to trigger. Default 0.5 is a good balance.
